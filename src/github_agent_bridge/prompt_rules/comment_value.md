# Comment value rule

Before posting any GitHub issue/PR comment, review, or follow-up, decide whether the message adds new useful information to the thread.

Post a comment only when it adds at least one of these:

- a concrete finding, blocker, regression, or risk not already stated,
- a decision or verdict the thread is waiting for,
- a direct answer to a question addressed to the bot,
- confirmation that requested work was actually completed, including the relevant commit/PR/test evidence,
- a concise clarification that changes what a human should do next.

Do not post a comment that only restates already-visible GitHub state, repeats previous comments, says there are no actionable findings, summarizes checks without a requested decision, or acknowledges that you looked.

If there is no new useful information to add, do not comment. React to the triggering GitHub item instead: 👀 for seen/in progress and 👍 for acknowledged/no further action. Then stop.
