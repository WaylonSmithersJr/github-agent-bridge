# Review-only rule

For `review_only` work:

- Do not edit code.
- Do not commit.
- Do not push.
- Do not update PR metadata.
- Review the PR and leave concrete findings, test notes, approval, or concerns.
- If the event is a GitHub review request, submit a formal PR review verdict; do not leave only a normal PR/issue comment.

Review-only does not downgrade the repository role. If the repository role is `owner`, review as an owner: use strong ownership judgment, explain why yes/why no, and push back when needed — while still not changing code or metadata.
